Activity

1. 앱 한 화면 입니다.
2. 첫 화면에서 다른 화면으로 넘어 간다는 것은 Activity가 바뀌는 것이다
3. Life cycle(수명 주기)가 존재한다 (생성과 소멸이 있다).
4. 콜백 메서드가 특정한 동작이 발생 되면 알려 준다.

onCreate
: activity가 만들어질 때 단 한번만 호출 된다.
: 단 한번 만 해야 되는 작업을 여기서 해주면 된다.

onStart
:호출되면 화면이 사용자에게 표시되고 화면이 포그라운드에 보내져서
사용자와 상호작용을 할수 있도록 준비한다.(eventListner같은걸 준비함)

onResume
:화면이 보여졌을때 호출된다.
:다시 앱으로 돌아올 때 무조건 호출 된다
:activity가 다시 호출 될 때  하면 되는 작업들을 여기서 해주면된다.
(메모리에 다시 올려야하는 객체들이있으면 여기서 처리해주면된다)

onPause
:화면에 일부가 가려 졌을 때
(화면을 검은색으로 바꿔라??같은 기능을 여기서 처리)

onStop
:화면이 전부가 보이지 않을 때
(임시 데이터들을 저장해 놓을 수 있다)
(사용자의 편의성을 위해서 저장을 해놨다가 다시 앱이 onResume 됬을때 저장했던 파일을 다시 불러오게 처리 할수있다)

onDestroy
:activity가 완전히 소멸 되기 전에 호출 된다.
:사용자가 화면을 닫거나 종료 기능을 눌렸을때 호출된다

구성변성(기기 화면 또는 멀티 윈도우 모드)로 인해 시스템에서
일시적으로 activity를 소멸 시키는 경우
